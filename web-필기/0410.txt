 +테이블 테그

 -<table>
 -<tr>,<th>,<td>

 <caption>,<thread>,<tbody>,<tfoot>

 -표를 표현하는 태그
 -table:표를 만들때 사용하는 태그,자식태그들로 행과 열을 표현한다
 -tr:table row,테이블의 각 행을 표현한다
 -th:table head,테이블의 제목을 나타낸다(한칸,한개의 열)
 -td :table data,태이블의 데이터를 나타낸다(한칸,한개의 열)

	<table>
	<tr>
	    <th></th>
	    <th></th>
	</tr>

	<tr>
	    <td></td>
	    <td></td>
	</tr>

	...
	</table>

 -thead: table header,테이블의 최상단행으로 고정 시키는 태그

 -tfoot: table Footer,행을 테이블 내부에서 최하단으로 고정시키는 태그

 -tbody:table body,행들을 thead,tfoot사이에 위치하도록 설정하는 태그

 -caption:테이블의 이름, 간단한 설명을 작성한다
	(테이블의 상단 바깥쪽에 위치한다)

 +form 태그
 -클라이어트가 서버로 요청을 보낼때
 데이터를 포함시켜주는 태그

 -요청 데이터를 입력할수 있게 화면을 구성해준다
	**요청 파라미터,전달 파라미터 : 요청데이터

 -action 속성, method속성을 필수로 작성해야한다
	
	action 속성: 요청 URL을 지정한다

	method 속성: 요청 방식을 지정한다
	요청 데이터 전달 방식
	get:
	post


 +http의 요청 방식(리퀘스트 메소드)
 -Http요청을 서버로 보낼때 데이터를 전달하는 방식
 - 요청 메시지에는 반드시 요청 방식을 지정해 주어야한다

 -get method
	데이터를 쿼리스트링으로 url마지막에 ?를 붙이고 그 뒤에 작성한다
	**쿼리스트링
	http통신에서 전달 데이터의 변수명와 값을 표현하는 형식
	변수명=데이터 쌍으로 표현
	여러개의 변수는 &로 구분한다
 -post method
	전달데이터를 쿼리스트링으로 요청메시지의 body영역에 포함시켜 전달하는 방식

 **get메소드로 요청을 보내면 전달데이터가 url에 보인다
 **post메소드로 요청을 보내면 전달데이터가 안보인다
 **전달데이터없이 url요청을 보낼때 get방식을 사용한다
